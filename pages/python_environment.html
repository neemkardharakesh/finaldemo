Python Environment

Python virtual environments are highly recommended when working with EXOSUITES. Virtual environments isolate project dependencies, prevent package conflicts, and make development significantly easier.

While EXOSUITES can be installed directly into the system Python environment, we strongly recommend using either Conda or venv for managing packages and dependencies.

Why Use Virtual Environments?

  • Avoid dependency conflicts.
  • Keep projects isolated.
  • Easy package management.
  • Simple environment recreation.
  • Safer software development.

Conda

Conda is one of the most popular tools for managing Python environments. Unlike standard virtual environments, Conda allows users to easily select Python versions and manage complex dependencies.

Recommended Conda distributions:

  • Miniforge (Recommended)
  • Miniconda
  • Micromamba
  • Anaconda

Installing Miniforge

Download the installer corresponding to your system architecture.

For Raspberry Pi and Jetson Orin Nano, use: 

```

aarch64
```

Install using: 

```

bash Miniforge3-$(uname)-$(uname -m).sh
```

Creating a Conda Environment

Create a new environment: 

```

conda create -n exosuites python=3.10
```

Activate the environment: 

```

conda activate exosuites
```

Deactivate the environment: 

```

conda deactivate
```

Installing EXOSUITES

Once the environment is active: 

```

pip install exosuites
```

All required package dependencies will be installed automatically.

Installing Additional Packages

Conda environments fully support pip installations. 

```

pip install numpy
pip install scipy
pip install matplotlib
pip install opencv-python
```

Using venv

Python provides built-in virtual environment support through venv. This is lightweight and available on nearly all Python installations.

Installing venv

On Ubuntu-based systems: 

```

sudo apt install python3-venv
```

Creating a venv Environment

Create a virtual environment: 

```

python3 -m venv venv
```

This creates a folder named: 

```

venv/
```

Activating venv

Activate the environment: 

```

source venv/bin/activate
```

After activation, your terminal prompt will display: 

```

(venv)
```

Deactivating venv

```

deactivate
```

Recommended Setup

💡 Recommendation

For EXOSUITES development we recommend:

  • Miniforge
  • Python 3.10 or newer
  • Dedicated Conda environment
  • pip installation inside the environment

Verification Checklist

  • Python 3.9+ installed.
  • Virtual environment created.
  • Environment activated.
  • EXOSUITES installed.
  • Dependencies installed successfully.
  • Ready for development.
